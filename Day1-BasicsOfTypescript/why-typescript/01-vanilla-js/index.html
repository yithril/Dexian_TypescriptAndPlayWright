Why the receipt broke

Nobody wrote a bug. The shape of a "book" changed, and the old code never knew.

1 · The data changed shape

Old shape — a physical book

When every book shipped in a box.

{
  "format": "physical",
  "id": "PT-1001",
  "title": "The Silent Library",
  "price": 14.99,
  "weightKg": 0.45
}

New shape — an eBook

An eBook is a download — no weight, it has a file size.

{
  "format": "ebook",
  "id": "PT-3001",
  "title": "Down the Foggy Road",
  "price": 7.99,
  "weightKg": removed
  "fileSizeMb": 3.2
}
2 · Where the old code lands on it
function shippingCost(book) {
  return book.weightKg * 2.5;
}
physical book.weightKg is 0.45 → 0.45 * 2.5 = 1.13 ✓
eBook book.weightKg is undefined → undefined * 2.5 = NaN ✗
3 · And it flows straight to the screen
new eBook JSON → shippingCost(book) → NaN → Order total $NaN

JavaScript never flagged it. It ran the broken cart and rendered it. That is the landmine.
