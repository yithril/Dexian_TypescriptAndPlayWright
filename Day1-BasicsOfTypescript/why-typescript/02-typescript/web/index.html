Checkout · TypeScript

Same change, caught before it could break

The eBook was added. The receipt is correct. Here is what TypeScript did along the way.

1 · The data changed shape

Old shape — a physical book

When every book shipped in a box.

{
  "format": "physical",
  "id": "PT-1001",
  "title": "The Silent Library",
  "price": 14.99,
  "weightKg": 0.45
}

New shape — an eBook

An eBook is a download — no weight, it has a file size.

{
  "format": "ebook",
  "id": "PT-3001",
  "title": "Down the Foggy Road",
  "price": 7.99,
  "weightKg": removed
  "fileSizeMb": 3.2
}
2 · We wrote the contract down
interface PhysicalBook { format: 'physical'; price: number; weightKg: number; }
interface EBook       { format: 'ebook';    price: number; fileSizeMb: number; }

type Book = PhysicalBook | EBook;
explicit A Book is a physical book or an eBook. You cannot invent fields, and the ones that exist are guaranteed.
3 · The compiler forced us to handle the new case
function shippingCost(book: Book): number {
  switch (book.format) {
    case 'physical': return book.weightKg * 2.5;
    case 'ebook':    return 0;
    default:
      const _exhaustive: never = book;
      return _exhaustive;
  }
}
forget a case? Leave out 'ebook' and the build stops with a precise message, pointing at the exact line:
fulfillment.ts:6 - error TS2322: Type 'EBook' is not assignable to type 'never'.
4 · So it reaches the screen correct
new eBook JSON → typed as Book → shippingCost handles 'ebook' → shipping $0.00 → Order total $24.11

Same backend change as the JavaScript version. This time the compiler walked us to every spot that needed updating — nothing reached the customer broken.
