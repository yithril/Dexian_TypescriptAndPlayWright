How Playwright locators really work

A locator is a description of how to find an element - resolved fresh every time it is used, not a captured reference. Each scenario makes one point.

  • Scenario 1: appears late The target is not in the DOM at load. It is injected after a random delay. The assertion auto-waits - no explicit wait.
  • Scenario 2: destroy & re-create Remove the element, then add a brand-new one. The same locator re-resolves it - no stale-element re-targeting.
  • Scenario 3: becomes clickable The button starts disabled and enables after a delay. click() waits for actionability before acting.
